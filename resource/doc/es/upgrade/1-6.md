```
composer config -g --unset repos.packagist
composer require -W workerman/webman-framework ^v1.6.11
```

# webman-framework 1.6.0
## Nuevas funciones

Requiere `PHP>=8.0`

### Soporte para obtener entradas mediante parámetros del controlador
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
Acceda a `/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` para obtener el resultado
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

Los parámetros también soportan vinculación a clases incluyendo modelos
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

Consulte [Vinculación de parámetros del controlador](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### Soporte de middleware de controlador
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### Soporte para Route::fallback()->middleware(...) en peticiones 4xx
Normalmente las peticiones 404 no pasan por ningún middleware. Desde la versión 1.6.0 puede configurar middleware para peticiones 4xx
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### Soporte para Route::disableDefaultRoute()
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[Documentación](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### Soporte para $request->setGet() $request->setPost() $request->setHeader()
```php
$request->get(); // Suponga ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // Resultado final ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[Documentación](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() admite omitir el parámetro de plantilla y rutas absolutas
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

#### Otros cambios de configuración
La configuración listen de config/server.php se movió a config/process.php. Solo afecta a proyectos nuevos.
El directorio process se movió a app/process. Solo afecta a proyectos nuevos.

## Notas de actualización

Si el código no era muy estándar, pueden surgir problemas de compatibilidad:

#### Verificar uso de view()
Compruebe si view() usa parámetros de plantilla que empiezan por `/`. Si es así, elimine el `/`
```
return view('/user/index');
// Cambiar a
return view('user/index');
```
#### Verificar coherencia de parámetros de rutas personalizadas
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// Cambiar a
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
Nota: Añadir type hint Request antes del parámetro $request.

#### Desactivar proxy de Composer
Los espejos de Composer como Aliyun tienen retrasos. Ejecute antes de actualizar:

```
composer config -g --unset repos.packagist
```
