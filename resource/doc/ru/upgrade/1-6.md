# Руководство по обновлению 1.6

**Перед обновлением сделайте резервную копию и выполните следующие команды**
```
composer config -g --unset repos.packagist
composer require -W workerman/webman-framework ^v1.6.11
```

# webman-framework 1.6.0
## Новые возможности

Требуется `PHP>=8.0`

### Поддержка получения входных данных через параметры контроллера
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
Доступ к `/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` даёт результат
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

Параметры также поддерживают привязку к классам, включая модели
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

См. [Привязка параметров контроллера](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### Поддержка middleware контроллера
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### Поддержка Route::fallback()->middleware(...) для запросов 4xx
Обычно запросы 404 не проходят через middleware. С версии 1.6.0 можно задать middleware для запросов 4xx
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### Поддержка Route::disableDefaultRoute()
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[Документация](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### Поддержка $request->setGet() $request->setPost() $request->setHeader()
```php
$request->get(); // Например ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // Итог ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[Документация](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() поддерживает опускание параметра шаблона и абсолютные пути
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

#### Прочие изменения конфигурации
Настройки listen из config/server.php перенесены в config/process.php. Затрагивает только новые проекты.
Каталог process перенесён в app/process. Затрагивает только новые проекты.

## Заметки по обновлению

Если код проекта не был достаточно стандартизирован, возможны проблемы совместимости:

#### Проверить использование view()
Проверьте, использует ли view() параметры шаблона, начинающиеся с `/`. Если да, удалите начальный `/`
```
return view('/user/index');
// Изменить на
return view('user/index');
```
#### Проверить согласованность параметров пользовательских маршрутов
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// Изменить на
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
Примечание: Добавить подсказку типа Request перед параметром $request.

#### Отключить прокси Composer
Зеркала Composer вроде Aliyun имеют задержки. Выполните перед обновлением:

```
composer config -g --unset repos.packagist
```
