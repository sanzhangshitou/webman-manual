# Базовые плагины

Базовые плагины обычно представляют собой общие компоненты, которые, как правило, устанавливаются через Composer, а код размещается в каталоге vendor. При установке пользовательские настройки (middleware, процессы, маршруты и т. д.) можно автоматически скопировать в каталог `{основной проект}config/plugin`. webman автоматически распознает конфигурацию в этом каталоге и объединит её с основной конфигурацией, что позволит плагинам вмешиваться в любой этап жизненного цикла webman.

Подробнее см. [Создание базового плагина](create.md).
