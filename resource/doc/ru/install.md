# Как установить webman

* PHP >= 8.1
* [Composer](https://getcomposer.org/) >= 2.0


## Linux: Установка PHP + Composer (пропустить, если уже настроено)
```
curl -sO https://www.workerman.net/install-php-and-composer && sudo bash install-php-and-composer
```
> **Примечание**
> Команда выше применима к Linux и macOS. Пользователям Windows нужно установить PHP отдельно.

Вы также можете вручную скачать [статическую версию PHP](https://www.workerman.net/download), предоставленную webman, и распаковать её для использования.

## 1. Создание проекта

```php
composer create-project workerman/webman:~2.0
```

> **Подсказка**
> При ошибках возможно использование проблемного зеркала Composer. Выполните `composer config -g --unset repos.packagist`, чтобы отключить прокси.

## 2. Запуск

Перейдите в каталог webman

#### Пользователи Windows
Дважды щёлкните по `windows.bat` или запустите `php windows.php` для старта

> **Подсказка**
> При ошибках некоторые функции могут быть заблокированы. См. [Проверка запрета функций](others/disable-function-check.md) для разблокировки.

#### Пользователи Linux
**Режим отладки** (для разработки: вывод отображается в терминале; служба останавливается при закрытии терминала)

```php
php start.php start
```

**Режим демона** (для продакшена: вывод не отображается в терминале; служба продолжает работать после закрытия терминала)

```php
php start.php start -d
```

#### Пользователи Docker

Запуск всех сервисов с выводом в консоль
```php
docker-compose up
```

Запуск сервисов в фоновом режиме
```php
docker-compose up -d
```

> **Подсказка**
> При ошибках некоторые функции могут быть заблокированы. См. [Проверка запрета функций](others/disable-function-check.md) для разблокировки.

## 3. Доступ

Откройте в браузере `http://ip-адрес:8787`.
