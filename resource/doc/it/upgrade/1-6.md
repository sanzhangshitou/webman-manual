# webman-framework 1.6.0
## Nuove funzionalità

Richiede `PHP>=8.0`

### Supporto per l'acquisizione di input tramite parametri del controller
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
Accesso a `/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` per ottenere il risultato
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

I parametri supportano anche il binding con le classi inclusi i modelli
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

Vedere [Binding parametri controller](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### Supporto middleware del controller
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### Supporto per Route::fallback()->middleware(...) per richieste 4xx
Normalmente le richieste 404 non passano per alcun middleware. Dalla versione 1.6.0 è possibile impostare middleware per richieste 4xx
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### Supporto per Route::disableDefaultRoute()
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[Documentazione](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### Supporto per $request->setGet() $request->setPost() $request->setHeader()
```php
$request->get(); // Es. ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // Risultato finale ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[Documentazione](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() supporta l'omissione del parametro template e percorsi assoluti
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

#### Altri cambiamenti di configurazione
La configurazione listen di config/server.php è stata spostata in config/process.php. Interessa solo i nuovi progetti.
La directory process è stata spostata in app/process. Interessa solo i nuovi progetti.

## Note di aggiornamento

Se il codice non era molto standardizzato, potrebbero verificarsi problemi di compatibilità:

#### Verificare l'uso di view()
Controllare se view() usa parametri template che iniziano con `/`. In tal caso, rimuovere il `/`
```
return view('/user/index');
// Cambiare in
return view('user/index');
```
#### Verificare coerenza parametri route personalizzati
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// Cambiare in
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
Nota: Aggiungere type hint Request prima del parametro $request.

#### Disattivare proxy Composer
Gli specchi Composer come Aliyun hanno ritardi. Eseguire prima dell'aggiornamento:

```
composer config -g --unset repos.packagist
```
