# webman-framework 1.6.0
## Novas funcionalidades

Requer `PHP>=8.0`

### Suporte para obter entrada via parâmetros do controlador
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
Acesse `/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` para obter o resultado
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

Os parâmetros também suportam vinculação a classes incluindo modelos
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

Consulte [Vinculação de parâmetros do controlador](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### Suporte a middleware do controlador
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### Suporte para Route::fallback()->middleware(...) em requisições 4xx
Normalmente requisições 404 não passam por nenhum middleware. A partir da versão 1.6.0 pode definir middleware para requisições 4xx
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### Suporte para Route::disableDefaultRoute()
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[Documentação](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### Suporte para $request->setGet() $request->setPost() $request->setHeader()
```php
$request->get(); // Ex. ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // Resultado final ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[Documentação](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() suporta omitir parâmetro de template e caminhos absolutos
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

#### Outras alterações de configuração
A configuração listen de config/server.php foi movida para config/process.php. Afeta apenas projetos novos.
O diretório process foi movido para app/process. Afeta apenas projetos novos.

## Notas de atualização

Se o código não era muito padronizado, podem surgir problemas de compatibilidade:

#### Verificar uso de view()
Verificar se view() usa parâmetros de template que começam com `/`. Se sim, remover o `/`
```
return view('/user/index');
// Alterar para
return view('user/index');
```
#### Verificar consistência de parâmetros de rotas personalizadas
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// Alterar para
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
Nota: Adicionar type hint Request antes do parâmetro $request.

#### Desativar proxy do Composer
Espelhos do Composer como Aliyun têm atrasos. Execute antes da atualização:

```
composer config -g --unset repos.packagist
```
