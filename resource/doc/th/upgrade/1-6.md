
# webman-framework 1.6.0
## ฟีเจอร์ใหม่

ต้องใช้ `PHP>=8.0`

### รองรับการรับข้อมูลผ่านพารามิเตอร์คอนโทรลเลอร์
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
เข้าถึง `/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` เพื่อดูผลลัพธ์
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

พารามิเตอร์ยังรองรับการผูกกับคลาสรวมถึงโมเดล
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

ดู [การผูกพารามิเตอร์คอนโทรลเลอร์](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### รองรับ middleware ของคอนโทรลเลอร์
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### รองรับ Route::fallback()->middleware(...) สำหรับคำขอ 4xx
ปกติคำขอ 404 ไม่ผ่าน middleware แต่จากเวอร์ชัน 1.6.0 สามารถตั้ง middleware สำหรับคำขอ 4xx ได้
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### รองรับ Route::disableDefaultRoute()
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[เอกสาร](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### รองรับ $request->setGet() $request->setPost() $request->setHeader()
```php
$request->get(); // เช่น ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // ผลลัพธ์สุดท้าย ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[เอกสาร](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() รองรับการละพารามิเตอร์เทมเพลตและเส้นทางแบบสัมบูรณ์
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

#### การเปลี่ยนแปลง config อื่นๆ
config listen ของ config/server.php ย้ายไป config/process.php กระทบเฉพาะโปรเจกต์ใหม่
โฟลเดอร์ process ย้ายไป app/process กระทบเฉพาะโปรเจกต์ใหม่

## หมายเหตุการอัปเกรด

หากโค้ดเดิมไม่มาตรฐาน อาจเกิดปัญหาความเข้ากันได้:

#### ตรวจสอบการใช้งาน view()
ตรวจสอบว่า view() ใช้พารามิเตอร์เทมเพลตที่ขึ้นต้นด้วย `/` หรือไม่ ถ้ามี ให้ลบ `/` ด้านหน้า
```
return view('/user/index');
// เปลี่ยนเป็น
return view('user/index');
```
#### ตรวจสอบความสอดคล้องของพารามิเตอร์ route ที่กำหนดเอง
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// เปลี่ยนเป็น
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
หมายเหตุ: เพิ่ม type hint Request ก่อนพารามิเตอร์ $request

#### ปิด proxy ของ Composer
 mirror ของ Composer เช่น Aliyun มีความล่าช้า รันก่อนอัปเกรด:

```
composer config -g --unset repos.packagist
```
