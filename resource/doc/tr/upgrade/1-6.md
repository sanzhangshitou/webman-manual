
# webman-framework 1.6.0
## Yeni Özellikler

`PHP>=8.0` gerekir

### Kontrolör parametreleri ile girdi alma desteği
```php
<?php
namespace app\controller;
use support\Response;

class UserController
{
    public function create(string $name, int $age, float $balance, bool $vip, array $extension): Response
    {
        return json([
            'name' => $name,
            'age' => $age,
            'balance' => $balance,
            'vip' => $vip,
            'extension' => $extension,
        ]);
    }
}
```
`/user/create?name=tom&age=18&balance=100.5&vip=1&extension[foo]=bar` adresine erişerek sonucu alın
```
{
  "name": "tom",
  "age": 18,
  "balance": 100.5,
  "vip": true,
  "extension": {
    "foo": "bar"
  }
}
```

Parametreler model dahil sınıflara bağlamayı da destekler
```php
<?php
namespace app\controller;
use app\model\User;
class UserController
{
    public function create(User $user): int
    {
        $user->save();
        return $user->id;
    }
}
```

Bkz. [Kontrolör parametre bağlama](https://www.workerman.net/doc/webman/controller.html#controller-parameter-binding)

#### Kontrolör middleware desteği
```php
<?php
namespace app\controller;
use app\middleware\MiddlewareA;
use app\middleware\MiddlewareB;
use support\Request;
class IndexController
{
    protected $middleware = [
        MiddlewareA::class,
        MiddlewareB::class,
    ];
    public function index(Request $request): string
    {
        return 'hello';
    }
}
```

#### Route::fallback()->middleware(...) ile 4xx istekleri için middleware ekleme
Normalde 404 istekleri middleware'den geçmez. 1.6.0'dan itibaren 4xx istekleri için middleware ayarlanabilir
```php
Route::fallback(function(){
    return json(['code' => 404, 'msg' => '404 not found']);
})->middleware([
    app\middleware\MiddlewareA::class,
    app\middleware\MiddlewareB::class,
]);
```

#### Route::disableDefaultRoute() desteği
```php
Route::disableDefaultRoute();
Route::disableDefaultRoute('', 'admin');
Route::disableDefaultRoute('foo');
Route::disableDefaultRoute('foo', 'admin');
Route::disableDefaultRoute([\app\controller\IndexController::class, 'index']);
```

[Dokümantasyon](https://www.workerman.net/doc/webman/route.html#disable-default-routes)

#### $request->setGet() $request->setPost() $request->setHeader() desteği
```php
$request->get(); // Örn. ['name' => 'tom', 'age' => 18]
$request->setGet(['name' => 'tom']);
$request->get(); // Sonuç ['name' => 'tom']
$request->setPost();
$request->setHeaders();
```

[Dokümantasyon](https://www.workerman.net/doc/webman/request.html#override-parameters)

#### view() şablon parametresini atlama ve mutlak yolları destekler
```php
<?php
namespace app\controller;
use support\Request;
class UserController
{
    public function hello(Request $request)
    {
        return view(['name' => 'webman']);
    }
}
```

## Yükseltme Notları

Kod standart değilse uyumluluk sorunları olabilir:

#### view() kullanımını kontrol edin
view() şablon parametrelerinin `/` ile başlayıp başlamadığını kontrol edin. Varsa `/` kaldırın
```
return view('/user/index');
// Şu şekilde değiştirin
return view('user/index');
```
#### Özel route parametre tutarlılığını kontrol edin
```
Route::any('/user/{name}', function (Request $request, $myname) {
   return response($myname);
});
// Şu şekilde değiştirin
Route::any('/user/{name}', function (Request $request, $name) {
   return response($name);
});
```
Not: $request parametresinden önce Request tip ipucu ekleyin.

#### Composer proxy'sini devre dışı bırakın
Aliyun gibi Composer aynalarında gecikme var. Yükseltmeden önce çalıştırın:

```
composer config -g --unset repos.packagist
```
