# パッケージング

例えば、fooアプリケーションプラグインをパッケージ化する場合：

* `plugin/foo/config/app.php`内のバージョン番号を設定する（**重要**）
* `plugin/foo`内のパッケージに不要なファイルを削除する。特に`plugin/foo/public`下のアップロード機能テスト用の一時ファイルを削除する
* プロジェクトにデータベーステーブル作成などの操作が含まれる場合、`plugin/foo/install.sql`を適切に設定する。詳細は[データベース](database.md)を参照
* プロジェクトに独自のデータベース・Redis設定がある場合、これらの設定をまず削除する。これらはアプリケーション初回アクセス時にインストールウィザードで入力させる（自前で実装する必要あり）。管理者が手動で入力・生成できるようにする
* プロジェクトにwebman adminバックエンドメニューが含まれる場合、`plugin/foo/config/menu.php`を設定する。これによりプラグインインストール時にこれらのメニューが自動設定される。詳細は[webman-adminメニューインポート](https://www.workerman.net/doc/webman-admin/app-development/menu.html)を参照
* 元の状態に戻す必要がある他のファイルを復元する
* 上記の操作が完了したら、`{メインプロジェクト}/plugin/`ディレクトリに移動する
* Linuxユーザー：`zip -r foo.zip foo`コマンドを実行してfoo.zipを生成する
* Windowsユーザー：fooフォルダを右クリックし「ZIPファイルに圧縮」を選択してfoo.zipを生成する

**foo.zipがパッケージ後のファイルです。次の章[プラグインの公開](publish.md)を参照してください**
