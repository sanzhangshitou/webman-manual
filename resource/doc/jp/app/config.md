# 設定ファイル

プラグインの設定は通常の webman プロジェクトと同様に動作しますが、通常は対象プラグインにのみ適用され、メインプロジェクトには影響しません。
例えば、`plugin.foo.app.controller_suffix` の値はプラグインのコントローラ接尾辞にのみ影響し、メインプロジェクトには影響しません。
例えば、`plugin.foo.app.controller_reuse` の値はプラグインがコントローラを再利用するかどうかにのみ影響し、メインプロジェクトには影響しません。
例えば、`plugin.foo.middleware` の値はプラグインのミドルウェアにのみ影響し、メインプロジェクトには影響しません。
例えば、`plugin.foo.view` の値はプラグインが使用するビューにのみ影響し、メインプロジェクトには影響しません。
例えば、`plugin.foo.container` の値はプラグインが使用するコンテナにのみ影響し、メインプロジェクトには影響しません。
例えば、`plugin.foo.exception` の値はプラグインの例外処理クラスにのみ影響し、メインプロジェクトには影響しません。

ただし、ルーティングはグローバルであるため、プラグインが設定したルートもグローバルなルーティングに影響します。

## 設定の取得
プラグインの設定を取得するには、`config('plugin.{plugin}.{設定項目}');` を使用します。例えば、`plugin/foo/config/app.php` の全設定を取得するには `config('plugin.foo.app')` を使用します。
同様に、メインプロジェクトや他のプラグインでも `config('plugin.foo.xxx')` で foo プラグインの設定を取得できます。

## 非対応の設定
アプリケーションプラグインは、`server.php` および `session.php` の設定、ならびに `app.request_class`、`app.public_path`、`app.runtime_path` の設定には対応していません。
